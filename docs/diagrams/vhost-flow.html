NaxOne · 虚拟主机双写流程

表单提交 → 双层注入校验 → Nginx + Apache 配置同步 → hosts 写入 → reload

📝
用户提交表单START
CreateVhostRequest（server_name / aliases / port / document_root / php_version / ssl / rewrite_rule / custom_directives ...）
▼
🛡
vhost.validate()
域名 / 别名格式校验（hostname 字符）
index_files / access_log 单行字段（无 ; { }）
custom_directives 花括号配对、拒绝 server { 顶层块
📂
validate_document_root()
必须是绝对路径
禁止指向系统目录（C:\Windows、Program Files 等）
config_filename 用 sanitize_filename 防路径穿越
✗ 不合法 → ❌
✗ 系统路径 → ❌
🚫
注入防护拒绝HIGH-1
含 ; / 换行 / 大括号 → 错误：「域名格式不合法：xxx」
用户在前端看到 friendlyError 翻译后的可读消息
🚫
系统目录保护HIGH-2
document_root 指向 C:\Windows 等 → 拒绝：「不允许将网站目录设在系统路径下」
▼ 全部通过
📄
写 Nginx vhost.confAtomic
SimpleTemplateEngine 渲染 → tempfile + rename 原子替换 → 失败回滚已写入。生成 .bak 备份。
📄
写 Apache vhost.confDual-write
同样模板引擎渲染 Apache 格式 → 原子写。任一边失败时回滚另一边，保证 Nginx 和 Apache 配置始终一致。
▼
🌐
写 Windows hosts 文件UAC
追加 127.0.0.1 example.test 到 C:\Windows\System32\drivers\etc\hosts。普通权限会弹 UAC，临时 ps1 文件用不可预测 token + ACL 收紧。
🔄
Reload Nginx / Apacheif running
如 Web 服务器在跑：先 nginx -t 校验配置 → reload。配置错误时回滚所有改动（vhost.conf + hosts）。
▼ 全部成功
✅
站点创建完成DONE
前端列表新增条目 · Toast 通知「站点已创建，已自动 reload」 · emit services-changed 让其他页面刷新
